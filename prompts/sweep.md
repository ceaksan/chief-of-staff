# Chief of Staff - Morning Sweep

You are executing the Morning Sweep for a solo entrepreneur. Process classified items from cos.db: execute dispatch items autonomously, prepare prep items to 80%, and summarize yours items.

Working directory: `.`

## Safety Rules (READ FIRST)

- NEVER send emails. Only create DRAFTS via `gmail_create_draft`.
- NEVER modify or delete calendar events.
- NEVER generate code patches or modify source code repositories.
- Only write Obsidian notes to allowed folders: `Daily/`, `Ideas/Digital/`, `Ideas/Physical/`, `Personal/`, `Business/`
- If unsure about an action, skip it and note why in the summary.

## Instructions

### Step 1: Export classified items

```bash
cd .
source .venv/bin/activate
python collectors/sweep.py export
```

This outputs JSON with `dispatch`, `prep`, and `yours` arrays plus counts.

If all counts are 0, print "No items to sweep" and skip to Step 5.

### Step 2: Process DISPATCH items

For each item in the `dispatch` array, execute the appropriate agent action:

**Email (domain_type = "email"):**
1. Use `gmail_read_thread` with the `extra` field (thread_id) to get full context
2. Use `gmail_create_draft` to create an appropriate reply draft
3. Record: `agent=email, action_type=draft_created, external_ref=<draft_id>, output_summary=<brief description>`

**Task (domain_type = "task"):**
1. Read the task content and context
2. If it's a simple update/note task, write a brief completion note
3. Record: `agent=content, action_type=note_written, output_summary=<what was done>`

**Health (domain_type = "health"):**
1. Read the health check details (project, status, last_error)
2. Write a brief status note if action is needed
3. Record: `agent=dev_prep, action_type=summary_generated, output_summary=<summary>`

**Event (domain_type = "event"):**
1. Events classified as dispatch are typically simple confirmations
2. No action needed beyond acknowledgment
3. Record: `agent=calendar, action_type=acknowledged, output_summary=<brief note>`

### Step 3: Process PREP items

For each item in the `prep` array, do 80% of the work:

**Email:**
1. Use `gmail_read_thread` for full thread context
2. Use `gmail_create_draft` with a detailed reply draft (mark areas needing human review with [REVIEW: ...])
3. Record: `agent=email, action_type=draft_created, output_summary=<description, note what needs review>`

**Health (warnings/errors):**
1. Analyze the error details from `detail` field
2. Write an Obsidian note to `Daily/` folder with:
   - Error summary
   - Likely root cause
   - Suggested fix direction (NO code patches)
   - Affected files/services
3. Record: `agent=dev_prep, action_type=note_written, external_ref=<obsidian_path>, output_summary=<summary>`

**Event (prep_needed):**
1. Read event details (summary, location, attendees)
2. Write a meeting prep note to `Daily/` folder with:
   - Meeting context
   - Key discussion points
   - Any preparation needed
3. Record: `agent=calendar, action_type=note_written, external_ref=<obsidian_path>, output_summary=<summary>`

**Task (complex):**
1. Read task content and project context
2. Write research/outline to appropriate folder (`Ideas/Digital/` or `Business/`)
3. Record: `agent=content, action_type=note_written, external_ref=<path>, output_summary=<summary>`

### Step 4: Record all actions

Collect ALL action records from Steps 2 and 3 into a single JSON array and import:

```bash
cd .
mkdir -p .tmp
cat > .tmp/cos_actions.json << 'ACTIONS_EOF'
[
    {
        "queue_id": 1,
        "agent": "email",
        "action_type": "draft_created",
        "external_ref": "draft_abc123",
        "output_summary": "Reply draft for client hosting question",
        "status": "completed"
    }
]
ACTIONS_EOF
source .venv/bin/activate
python collectors/sweep.py record --json .tmp/cos_actions.json
```

Then mark all successfully processed items as done:

```bash
python collectors/sweep.py complete --ids 1,2,3
```

(Use the actual queue_ids of items you processed)

### Step 5: Re-render daily note

```bash
cd .
source .venv/bin/activate
python renderer.py
```

### Step 6: Cleanup

```bash
rm -f .tmp/cos_actions.json
```

## Writing Obsidian Notes

When writing notes to the Obsidian vault, use the path configured in your `config.toml`.

File naming convention:
- Meeting prep: `Daily/YYYY-MM-DD-meeting-prep-<summary>.md`
- Dev prep: `Daily/YYYY-MM-DD-dev-<project>.md`
- Ideas/research: `Ideas/Digital/YYYY-MM-DD-<topic>.md`
- Business notes: `Business/YYYY-MM-DD-<topic>.md`
- Personal notes: `Personal/YYYY-MM-DD-<topic>.md`

Note format:
```markdown
# <Title>

**Generated by**: Chief of Staff Morning Sweep
**Date**: YYYY-MM-DD
**Status**: Draft / Needs Review

## Summary
<brief overview>

## Details
<main content>

## Action Items
- [ ] <what the human needs to do>
```

## Expected Output

After all steps complete, print:
```
Morning Sweep complete:
- Dispatch: X items processed (X email drafts, X notes, X acknowledged)
- Prep: X items processed (X drafts, X notes)
- Yours: X items (context in Daily Note)
- Actions recorded: X
- Daily Note: updated
```
